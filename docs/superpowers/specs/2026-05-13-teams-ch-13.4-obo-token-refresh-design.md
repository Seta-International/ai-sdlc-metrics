## Goal

When Teams completes SSO sign-in for a user, exchange the SSO token for a Graph OBO token and store it in `oauth.oauth_tokens`. The mock handler continues returning hardcoded cards ‚Äî OBO just ensures the token vault is populated so 13.5 (the real Planner handler) can retrieve a valid Graph token without any additional auth ceremony.

---

## How Teams SSO triggers OBO

Teams sends a `signin/tokenExchange` invoke activity automatically on the user's first interaction, provided the manifest includes `webApplicationInfo`. The sequence:

```
1. User sends a message in Teams 1:1 chat with SetaAgent
2. Teams intercepts ‚Äî user not yet authenticated
3. Teams sends invoke { name: 'signin/tokenExchange', value: { token: <AAD SSO token> } }
4. sso.ts calls provider.acquireOnBehalfOf({ tenantId, userAssertion: ssoToken, scopes }) ‚Üí vault.put()
5. MSAL exchanges SSO token ‚Üí OBO access token + refresh token
6. Tokens upserted into oauth.oauth_tokens (single-flight, SELECT ‚Ä¶ FOR UPDATE)
7. Bot returns InvokeResponse { status: 200 }  ‚Üê success signal to Teams
8. Teams re-sends the original user message
9. Mock handler responds with a scripted card as before
```

If step 4 fails (admin consent not yet granted), the bot returns `{ status: 412 }` and Teams shows a "Sign in" button.

### New: `modules/channels/teams/src/sso.ts`

```ts
import type { OAuthProvider, TokenVault } from '@seta/oauth'
import { logger } from '@seta/observability'
import type { Activity, OutboundActivity } from './handler.js'

const PLANNER_SCOPES = [
  'https://graph.microsoft.com/Tasks.ReadWrite',
  'https://graph.microsoft.com/User.Read',
]

export async function handleSsoTokenExchange(
  activity: Activity,
  tenantId: string,
  deps: { provider: OAuthProvider; vault: TokenVault },
): Promise<OutboundActivity> {
  const value = activity.value as Record<string, unknown> | undefined
  const userAssertion = typeof value?.token === 'string' ? value.token : undefined

  if (!userAssertion) {
    logger.warn({ conversationId: activity.conversation.id }, 'signin/tokenExchange missing token')
    return { type: 'invokeResponse', value: { status: 400 } }
  }

  try {
    const bundle = await deps.provider.acquireOnBehalfOf({
      tenantId,
      userAssertion,
      scopes: PLANNER_SCOPES,
    })

    const homeAccountId = bundle.meta.homeAccountId as string
    await deps.vault.put(tenantId, 'entra', `user:${homeAccountId}`, bundle)

    return { type: 'invokeResponse', value: { status: 200 } }
  } catch (err) {
    // Consent not yet granted ‚Äî Teams will show the "Sign in" prompt
    logger.warn({ err, tenantId }, 'OBO exchange failed ‚Äî likely missing admin consent')
    return { type: 'invokeResponse', value: { status: 412 } }
  }
}
```

## `OAuthProvider.acquireOnBehalfOf` contract

Defined in `platform/oauth/src/provider.ts`. Actual signature:

```ts
acquireOnBehalfOf(input: {
  tenantId:      string    // seta tenant id (uuid)
  userAssertion: string    // the Teams SSO token from signin/tokenExchange
  scopes:        string[]
}): Promise<TokenBundle>
```

Returns `{ accessToken, refreshToken, meta: { homeAccountId, ... } }`. Does NOT store ‚Äî caller is responsible for `vault.put()`.

---

## Single-flight guarantee

`vault.put` uses the MSAL `ConfidentialClientApplication` token cache (per-tenant LRU in `platform/oauth`). If Teams retries `signin/tokenExchange` before the first call completes, MSAL's internal locking prevents a duplicate MSAL round-trip. No additional locking needed in the channel layer.
