# EMS Business Reverse-Engineering Spike

This spike documents the combined `EMS` backend and `EMS-fe` frontend as one internal Employee Management System for SETA International. It focuses on the business behavior already present in the repositories so a team can clone the current system faithfully, not redesign it.

## Documents

- `01-business-overview.md`: system purpose, domain, actors, scope, terminology, and confidence-rated conclusions.
- `02-capabilities-and-use-cases.md`: the main business capabilities and the user-facing use cases they support.
- `03-workflows-rules-and-domain.md`: end-to-end workflows, business rules, permissions, state transitions, and operational mechanics.
- `04-clone-critical-understanding.md`: the business behaviors, rules, actors, entities, and dependencies that cannot be lost if the system is cloned.

## Recommended Reading Order

1. `01-business-overview.md`
2. `02-capabilities-and-use-cases.md`
3. `03-workflows-rules-and-domain.md`
4. `04-clone-critical-understanding.md`
