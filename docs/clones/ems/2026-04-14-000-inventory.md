## Roles (Legacy)

7 roles across 3 scope types (Organization, Account, Project):

| Role             | Scope                   | Capabilities                                                 |
| ---------------- | ----------------------- | ------------------------------------------------------------ |
| SUPER_ADMIN      | Organization            | Full system access — all modules, all scopes                 |
| HR               | Organization            | Employee profiles, recruitment, project staffing, dashboards |
| EXECUTIVE        | Organization            | Oversight, accounts/projects, dashboards, read-only profiles |
| ACCOUNT_MANAGER  | Account (can be Org)    | Manage specific accounts, projects within account, staffing  |
| PROJECT_MANAGER  | Project (can be Org)    | Create/update projects, manage staffing, project dashboards  |
| EMPLOYEE         | Org / Account / Project | Own profile, view assigned project info                      |
| EXTERNAL_PARTIME | Org / Account / Project | Limited access, own profile, assigned project info           |

**Access control pattern:** JWT middleware → `RequireRole` FastAPI dependency with scope matching. Special bypasses: `RequireRoleForContractModule` (contract handlers bypass), `RequirePartnerPermission` (partner/webhook access).

**Scope inheritance:** Organization roles cascade to Account and Project scopes. Account roles override Org for that account. Project roles override Account for that project.
