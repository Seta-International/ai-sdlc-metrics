# Clone Brief: contract

## Business Purpose

Employment contract management is the legal backbone of HR operations — it defines the employment relationship (terms, compensation, duration, obligations) and generates the legal documents that formalize it. The legacy EMS has a monolithic 47KB contract service that handles everything from version lifecycle to DOCX generation to evaluation workflows.

This module is **decomposed across 3 target modules** following DDD boundaries:

- **`people`** — contract version lifecycle (create, activate, amend, extend, renew, expire, terminate)
- **`documents`** — template management, DOCX generation engine (docxtemplater), placeholder resolver, generated document storage
- **`admin`** — contract type configuration, jurisdiction rules

Pre-renewal evaluation moves to `performance` module (triggered by events). Contract handler RBAC bypass is replaced by kernel authority delegation.

## Decisions

| Feature                     | Decision      | Rationale                                                                                                                                                                                              |
| --------------------------- | ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Contract version lifecycle  | **reimagine** | Target has stub `contract_version`. Enhance with full version chain, snapshot terms (JSONB), change types (initial/amendment/extension/renewal), status machine. Industry standard from Personio/Deel. |
| Contract amendments         | **new**       | Record which fields changed + why per version. Store full terms snapshot + delta. Critical for payroll integration and audit.                                                                          |
| Expiry scanning + alerts    | **reimagine** | Legacy APScheduler → pg-boss daily job. Emit `ContractExpiringSoon` events at configurable thresholds (90/60/30/14/7 days). Computed `expiring` status, not persisted.                                 |
| Contract type configuration | **reimagine** | Legacy simple name-only lookup. Enhance: has_end_date, has_probation, default_probation_days, max_renewals, notice_period_days, auto_renewal. Vietnam jurisdiction rules as seed data.                 |
| DOCX template engine        | **reimagine** | Move to `documents` module. Python docxtpl → Node.js docxtemplater. Schema-driven placeholder resolver via QueryFacades. Template versioning (immutable, pinned at generation time).                   |
| Placeholder data mapper     | **reimagine** | Legacy hardcodes field paths. New: schema-driven resolver — template declares expected variables, resolver fetches from modules via facades. Formatters for dates, currency, case.                     |
| Document generation         | **reimagine** | Move to `documents` module. DOCX → optional PDF (LibreOffice headless). Bulk generation via pg-boss. Generated docs stored in S3 with metadata in DB.                                                  |
| Pre-renewal evaluation      | **reimagine** | Move to `performance` module. Contract emits `ContractRenewalEvaluationRequested` → performance creates evaluation → `RenewalEvaluationCompleted` flows back. Deferred — not in this task set.         |
| Contract handler RBAC       | **skip**      | Anti-pattern (custom access control bypass). Replace with kernel role_permission delegation for contract operations.                                                                                   |
| Google Translate            | **skip**      | Overkill. Use separate templates per language (VN + EN). Locale-aware formatters for dates/currency.                                                                                                   |
| DOCX bookmark insertion     | **skip**      | Legacy-specific XML manipulation. Not needed with modern template approach.                                                                                                                            |
| Handover status lookup      | **skip**      | No real workflow. If needed, simple enum on contract version, not separate table.                                                                                                                      |
| E-signature integration     | **new (v2)**  | Port interface in documents module. DocuSign + Vietnamese provider. Deferred.                                                                                                                          |
| Auto-renewal rules          | **new (v2)**  | Configurable per contract type. Requires evaluation integration. Deferred.                                                                                                                             |
| Contract expiry dashboard   | **keep**      | Counts (expiring today/week/month). Simple, useful for HR.                                                                                                                                             |

## Risks & Assumptions

- **docxtemplater syntax migration:** Legacy templates use Jinja2 `{{ }}` syntax. docxtemplater uses `{ }`. Existing DOCX templates must be migrated (placeholder syntax rewrite). Consider a one-time migration script.
- **Placeholder resolver complexity:** Schema-driven resolution requires all source modules to expose QueryFacade methods for contract-relevant data. People module must expose employee detail, finance must expose salary. Dependencies must be wired before document generation works end-to-end.
- **Vietnam jurisdiction rules:** Max 2 consecutive fixed-term contracts, probation max 60 days, conversion to indefinite after 36 months. These must be enforced at contract creation/renewal time, not just documented.
- **Terms snapshot size:** JSONB `terms` column stores full contract state per version. For typical contracts this is under 1KB per version — not a concern.
- **Template versioning:** When a template is updated, existing generated documents must not change. Immutable versions pinned at generation time. Storage cost is minimal (DOCX files are typically 50-200KB).
