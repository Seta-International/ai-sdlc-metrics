# Task: DOCX Generation Engine

## Scope

Implement the document generation engine in the `documents` module:

1. Render DOCX templates with resolved placeholder data using docxtemplater
2. Handle missing/unmapped values (draft mode vs final mode)
3. DOCX → PDF conversion via LibreOffice headless
4. Bulk generation via pg-boss
5. Document preview (render with data, return presigned URL)

## Roles Covered

- **HR:** Generate documents (single + bulk), preview documents
- **EMPLOYEE:** Download own generated documents

## Business Context

Document generation is the bridge between structured HR data and legal employment documents. The engine takes a DOCX template + resolved placeholder values and produces a personalized document. This replaces the legacy Python docxtpl + docx_filler.py system.

## Source Reference

- **Files:** `src/core/services/contract_service.py` (save_contract_document, generate_contract_preview, fill_placeholders), `src/utils/docx_filler.py`
- **Key logic:** Legacy fills DOCX template via docxtpl, highlights unmapped values in yellow, adds XML bookmarks. Uploads result to MinIO.

## Target Location

- **Where:** `apps/api/src/modules/documents/application/commands/`, `apps/api/src/modules/documents/infrastructure/`
- **Conventions to follow:** `@future/storage` for S3, pg-boss for async jobs

### Infrastructure

docxtemplater rendering:

```typescript
import Docxtemplater from 'docxtemplater';
import PizZip from 'pizzip';

async renderDocument(templateBuffer: Buffer, data: Record<string, any>, mode: 'draft' | 'final'): Promise<Buffer> {
  const zip = new PizZip(templateBuffer);
  const doc = new Docxtemplater(zip, {
    paragraphLoop: true,
    linebreaks: true,
    nullGetter(part) {
      if (mode === 'final') throw new MissingVariableError(part.value);
      return `[MISSING: ${part.value}]`;
    }
  });
  doc.render(data);
  return doc.getZip().generate({ type: 'nodebuffer' });
}
```

PDF conversion (infrastructure adapter):

```typescript
// LibreOffice headless or Gotenberg sidecar
```

## Edge Cases

- **Template not found:** 404 with clear error
- **Version mismatch:** If specific version requested doesn't exist, reject. If no version specified, use current_version.
- **Large batch (500+ items):** Chunk into sub-batches of 50 for meaningful progress updates
- **PDF conversion failure:** Mark item as completed_with_errors, keep DOCX (PDF is optional enhancement)
- **Concurrent generation:** Multiple users generating from same template simultaneously is fine (stateless rendering)
- **Template caching:** Cache template buffers per (tenantId, templateId, version) in memory/Redis to avoid repeated S3 fetches during batch processing
