**Status:** refined

## What This Module Does

The work-time module owns the per-user working schedule configuration. Each user can have multiple `work_time` rows covering non-overlapping date ranges. The "active" row is the one with the latest `from_date` where `to_date IS NULL` or `to_date >= today`.

Core responsibilities:

1. Store and manage per-user work schedule (shift start/end, break start/end, effective date range)
2. Enforce non-overlapping date ranges per user (validated via range overlap check)
3. When creating a new row, automatically close the previous open-ended row by setting `to_date = new fromDate - 1`
4. On create or update, emit a `WorkTimeConfigChanged` domain event so the attendance module can cascade-recalculate affected timesheet rows (BR-09)
5. Provide the pure calculation functions (`getLate`, `getEarly`, `getLack`, `getWorkTime`, `getOverTime`) as a shared value-object within the time module — used by both work-time cascade recalculation and attendance's check-in logic

| `getWorkTimeService` (list all rows for user, ordered desc)                                 | **keep + reimagine** | Port to `GetWorkTimeHandler` query; add role check (staff/manager = own userId only, hrm = any)                                                                                                                                                                                                                                                             |
| `updateWorkingTimeService` (partial update + cascade)                                       | **keep + reimagine** | Port to `UpdateWorkTimeHandler`; fix silent error swallow (legacy has empty catch — failures are invisible); trigger domain event on date/time field changes                                                                                                                                                                                                |
| `deleteWorkingTimeService`                                                                  | **keep + reimagine** | Port to `DeleteWorkTimeHandler`; straightforward delete with role guard                                                                                                                                                                                                                                                                                     |
| Duplicate/overlap check (`checkDuplicateWorkTimeQuery`)                                     | **fix + reimagine**  | Legacy uses raw string interpolation with user inputs (SQL injection). | `updateTimesheet` cascade (inline, forEach+async bug)                                       | **reimagine**        | Two bugs: (1) `forEach` with `async` callbacks doesn't await — cascade silently no-ops; (2) tight coupling — work-time directly updates timesheet rows. | Pure calculation functions (`getLate`, `getEarly`, `getLack`, `getWorkTime`, `getOverTime`) | **keep + reimagine** | Business math is correct and must be preserved exactly (BR-01: 6-min grace period = `6 * 60 * 1000 - 1` ms). Eliminate `moment` — replace with native Date arithmetic (duration math via ms arithmetic is already moment-agnostic in spirit). |
| `moment-timezone` dependency                                                                | **skip**             | Replace entirely with native Date arithmetic. The calculation functions only use `moment.duration()` for ms conversion and `moment.utc().format()` for HH:mm:ss — trivially replaceable                                                                                                                                                                     |
| `yesql` named SQL                                                                           | **skip**             | Drizzle ORM replaces all raw SQL                                                                                                                                                                                                                                                                                                                            |
| Role check missing on GET in legacy                                                         | **fix**              | `router.get(apiRoute.workTime)` has no `roleAndPermission` middleware — any authenticated user could query any user's work time by userId. New tRPC procedure enforces: staff/manager = own userId only; hrm = any userId                                                                                                                                   |

## Key Business Rules (must be preserved)

| Rule                                   | Implementation note                                                                                                                                   |
| -------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| BR-01: 6-min grace period              | `LATE_EARLY_GRACE_MS = 6 * 60 * 1000 - 1` constant in `AttendanceCalculator`                                                                          |
| BR-09: Work-time cascade recalculation | `CreateWorkTimeHandler` and `UpdateWorkTimeHandler` publish `WorkTimeConfigChanged` event; attendance module's `OnWorkTimeChangedHandler` picks it up |
| No overlapping ranges per user         | `WorkTimeRepository.findOverlapping(userId, tenantId, fromDate, toDate?, excludeId?)` — uses Drizzle `sql` with tstzrange overlap operator `&&`       |
| Close previous open-ended row          | `CreateWorkTimeHandler`: before insert, find latest row where `toDate IS NULL`, set `toDate = newFromDate - 1 day` in same transaction                |
| toDate nullable = open-ended           | `to_date` column is nullable; NULL means "current/ongoing"                                                                                            |

## Domain Event

```typescript
// packages/event-contracts/src/time/work-time-config-changed.event.ts
export interface WorkTimeConfigChangedEvent {
  type: 'time.WorkTimeConfigChanged'
  tenantId: string
  userId: string
  workTimeId: string
  fromDate: string // ISO date string
  toDate: string | null // ISO date string or null (open-ended)
  fromTime: string // HH:mm:ss
  toTime: string // HH:mm:ss
  startBreakTime: string // HH:mm:ss
  endBreakTime: string // HH:mm:ss
}
```
