# Module Brief: requests

**Source:** `timesheet-app` ŌĆö `server/services/request.service.js`, `server/query/request.query.js`, `server/routes/request.js`
**Target:** `apps/api/src/modules/time/` (sub-feature of the `time` module)
**Scope:** Forget requests + Compensation (late-early) requests only. Leave requests belong to the `leave` sub-module.
**Date:** 2026-04-14
**Status:** refined

---

## What This Module Does

Handles employee requests to correct attendance anomalies:

1. **Forget request** ŌĆö employee missed check-in and/or check-out. Requests that the timesheet be corrected with the provided timestamps.
2. **Compensation request** ŌĆö employee was late on `request_date` but worked overtime on `compensation_date`. The OT is used to offset the lack, and `timesheet.comp` is written to record this.

Both types share a two-step approval workflow: manager confirms ŌåÆ HRM approves/rejects.

## Key Domain Rules (must be enforced in handlers)

- **BR-02:** Max 3 forget requests per user per month (count `error_count = true`, status != rejected)
- **BR-03:** Max 6 distinct `request_date` days for comp per month; max 3 distinct `compensation_date` source-days per month (status != rejected)
- **BR-07:** On comp request create: read `timesheet.over_time` for compensation_date and `timesheet.lack` for request_date; write `min(over_time, lack)` ŌåÆ `timesheet.comp` for request_date; on update of dates: clear old comp, recalculate; on delete: clear comp
- **BR-11:** Manager query filters by `manager_id` only ŌĆö no `line_manager_id`
