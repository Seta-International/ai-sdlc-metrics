## Business Context

HRM uses this export for payroll processing and government reporting. The export includes work-schedule fields (`from_time`, `to_time`) alongside attendance data so the payroll team can compute salary deductions without needing another lookup. The legacy version has no row limit and queries the full date range in one shot — on a company with 200 employees and a 30-day range that is 6,000 rows max, which is manageable. Still, we add a 31-day hard cap for safety.

**Role:** hrm only. Kernel permission: `time:attendance:export`.

**Notable legacy query features to preserve:**

- Joins `work_time` to include `from_time` and `to_time` per day (work schedule effective at that date)
- Also fetches `off_time_hour` from approved `request` rows (leave hours taken)
- Filters: `wt.from_date <= t.date AND (t.date <= wt.to_date OR wt.to_date IS NULL)` — handles open-ended work schedules

## Handler Logic

```ts
async execute(query: ExportTimesheetQuery): Promise<TimesheetExportRow[]> {
  // 1. Enforce 31-day cap
  const from = new Date(query.fromDate)
  const to = new Date(query.toDate)
  const daysDiff = Math.ceil((to.getTime() - from.getTime()) / 86_400_000)
  if (daysDiff > 31) {
    throw new BadRequestException('Export date range cannot exceed 31 days')
  }
  if (daysDiff < 0) {
    throw new BadRequestException('fromDate must be before toDate')
  }

  // 2. Fetch raw rows from repository
  const rows = await this.timesheetRepo.findAllForExport(
    query.tenantId,
    query.fromDate,
    query.toDate,
    query.badgeNumber,
  )

  // 3. Enrich with people data (name, badgeNumber, managerName)
  //    Batch: collect unique userIds, call PeopleQueryFacade once
  const uniqueUserIds = [...new Set(rows.map(r => r.userId))]
  const profiles = await this.peopleFacade.getEmployeeProfiles(uniqueUserIds, query.tenantId)
  const profileMap = new Map(profiles.map(p => [p.actorId, p]))

  // 4. Map to export shape
  return rows.map(row => {
    const profile = profileMap.get(row.userId)
    return {
      day: row.date,
      userId: row.userId,
      badgeNumber: profile?.employeeCode ?? null,
      employeeName: profile?.displayName ?? null,
      managerName: null,  // TODO: enrich managerName via PeopleQueryFacade.getManagerName if needed
      // ... rest of fields mapped from row
    }
  })
}
```

## tRPC Procedure

Add under the `time.attendance` sub-router:

```ts
export: protectedProcedure
  .meta({ permission: 'time:attendance:export' })
  .input(z.object({
    fromDate: z.string().regex(/^\d{4}-\d{2}-\d{2}$/),
    toDate: z.string().regex(/^\d{4}-\d{2}-\d{2}$/),
    badgeNumber: z.string().optional(),
  }))
  .query(({ ctx, input }) =>
    svc().query(new ExportTimesheetQuery(ctx.tenantId, ctx.actorId, input.fromDate, input.toDate, input.badgeNumber))
  ),
```

**Note:** The tRPC procedure returns the array; the frontend is responsible for CSV serialization (using `papaparse` or similar). This avoids streaming complexity on the API side.
