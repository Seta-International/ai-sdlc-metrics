# Module Brief: attendance

**Source:** `timesheet-app` — `server/services/timesheet.js`, `server/query/timesheet.js`, `server/utils/timesheet.js`  
**Target:** `apps/api/src/modules/time/` (attendance sub-domain)  
**Date:** 2026-04-14  
**Status:** refined

---

## What This Module Does

The attendance module is the core of the timesheet system. It owns one row per user per calendar day (`time_sheet` table) and is responsible for:

1. Recording check-in/check-out (biometric writes come from external engine; online check-in is handled here)
2. Computing late, early, lack, work_time, and over_time using the user's configured work schedule
3. Providing paginated views of timesheet data at three scopes: own (staff), team (manager), and company-wide (manager+hrm)
4. Exporting timesheet data as CSV (hrm only)
5. Cascading recalculation of computed fields when work-time config changes (BR-09)

## Feature Decisions

| Feature                                                                 | Decision             | Rationale                                                                                                                                                                                                                                                                |
| ----------------------------------------------------------------------- | -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Timesheet domain entity + repository interface                          | **keep**             | Clean domain boundary; all computed fields map directly to `time_sheet` columns                                                                                                                                                                                          |
| Late/early calculation logic (`utils/timesheet.js`)                     | **keep + reimagine** | Business math is correct and must be preserved exactly (BR-01: 6-min grace). Port to a pure TS value-object / service; eliminate `moment` — replace with native Date arithmetic                                                                                          |
| `getMyTimesheetService`                                                 | **keep + reimagine** | Port to tRPC query handler. Generate-series pattern gives calendar rows even for days with no check-in; keep this behavior but express via Drizzle                                                                                                                       |
| `getMemberTimesheetService`                                             | **keep + reimagine** | Port to tRPC query handler. Remove `line_manager_id` join (BR-11). Look up manager's team via `PeopleQueryFacade` instead of cross-schema join to `user`                                                                                                                 |
| `getAllTimesheetService`                                                | **keep + reimagine** | Port to tRPC query handler. Remove cross-schema `user` join; use PeopleQueryFacade                                                                                                                                                                                       |
| `getATimesheetService`                                                  | **keep**             | Simple point lookup; trivial port                                                                                                                                                                                                                                        |
| `exportTimesheetService`                                                | **keep + reimagine** | Port to tRPC query. Keep CSV column set (adds work_time schedule columns). No streaming in legacy — keep as array response; frontend handles CSV serialization                                                                                                           |
| `checkInOnlineService`                                                  | **keep + reimagine** | Significant redesign: (1) idempotency guard (already checked-in → return existing row) is correct — keep it; (2) RRule validation is correct (BR-08) — port it; (3) email notification → replace with domain event `AttendanceViolationDetected`; (4) eliminate `moment` |
| Email notification in `sendMailNotification`                            | **reimagine**        | Direct `sendMailService` call is a tight coupling anti-pattern. Replace with domain event `AttendanceViolationDetected` published to `outbox_event`; the notification module consumes it                                                                                 |
| Recalculation triggered by work-time changes (BR-09)                    | **keep + reimagine** | Port the math; trigger via domain event `WorkTimeConfigChanged` consumed by an event handler in the attendance module                                                                                                                                                    |
| `line_manager_id` dual-manager query                                    | **skip**             | BR-11: single `manager_id` only                                                                                                                                                                                                                                          |
| `moment-timezone` dependency                                            | **skip**             | Replace entirely with native Date + date-fns/temporal. No moment in Future                                                                                                                                                                                               |
| `yesql` named SQL                                                       | **skip**             | Drizzle ORM replaces raw SQL                                                                                                                                                                                                                                             |
| Biometric check-in/out routes                                           | **skip**             | External device integration — separate concern, not in scope                                                                                                                                                                                                             |
| SQL injection risk in `countAllTimesheetQuery` / `getAllTimesheetQuery` | **fix**              | Legacy interpolates `name` and `badgeNumber` directly into SQL strings. Drizzle parameterized queries eliminate this entirely                                                                                                                                            |
| Missing pagination on export                                            | **fix**              | Legacy export has no limit — can return millions of rows. Add date-range cap (max 31 days) in the handler                                                                                                                                                                |

## Key Business Rules (must be preserved)

| Rule                                    | Implementation note                                                                              |
| --------------------------------------- | ------------------------------------------------------------------------------------------------ |
| BR-01: 6-min grace period               | `lateEarlyLimitMs = 6 * 60 * 1000 - 1` — keep exact constant in `AttendanceCalculator`           |
| BR-08: Online check-in RRule validation | Port `rrule` library usage; `inAccordance = listSchedule.length === 0` logic correct             |
| BR-09: Work-time cascade recalculation  | Event handler `OnWorkTimeChangedHandler` queries all active timesheets for user and recalculates |
| BR-11: Single manager_id                | Remove all `line_manager_id` joins; use `manager_id` only via PeopleQueryFacade                  |

| Dependency                                         | How resolved                                                                                 |
| -------------------------------------------------- | -------------------------------------------------------------------------------------------- |
| User profile (name, badge number, manager_id)      | `PeopleQueryFacade.getEmployeeProfile(actorId, tenantId)`                                    |

| Member schedule (RRule)                            | `member-schedule` sub-module within `time` module — direct import allowed                    |
| Violation notification                             | Domain event `AttendanceViolationDetected` → outbox → notification consumer                  |
