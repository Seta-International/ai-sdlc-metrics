## Key Business Rules

These must be preserved exactly in the migration:

| #     | Rule                                                                                                                                     |
| ----- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| BR-01 | Late/early grace period = **6 minutes** (≤ 6 min is not counted as late or early)                                                        |
| BR-02 | Max forget requests per user per month = **3**                                                                                           |
| BR-03 | Max compensation request-days per month = **6**; max compensation source-days = **3**                                                    |
| BR-04 | Carry-over leave is usable Jan–March only; requests outside Q1 are rejected                                                              |
| BR-05 | Leave unit = **hours** (8 h = 1 day); `off_time_hour` is calculated from work schedule                                                   |
| BR-06 | Leave balance is deducted on `status = 'approved'`, not on request creation                                                              |
| BR-07 | Comp request links `over_time` from compensation_date ↔ `lack` from request_date; writes `timesheet.comp`                                |
| BR-08 | Online check-in validates against user's RRule schedule; violation triggers notification to user + manager + admin                       |
| BR-09 | Changing a user's work-time config cascades a recalculation of all linked timesheet rows (late/early/lack/work_time/over_time)           |
| BR-10 | User provisioning auto-creates a default `work_time` row and a `user_leave` row (excluded from time module — belongs to people/identity) |
| BR-11 | Single manager structure: `manager_id` only (no `line_manager_id`)                                                                       |

## DB Tables (source schema, public)

| Table                | Description                                                                                                                                                                                                                                       | Owned by module |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------- |
| `time_sheet`         | One row per user per day; stores check-in/out, late, early, lack, work_time, over_time, in_office, comp, holiday_id, in_accordance, work_schedule, check_in_type, actual_in/out                                                                   | attendance      |
| `work_time`          | Per-user working schedule config with date ranges (from_date, to_date) and break times                                                                                                                                                            | work-time       |
| `holiday`            | Company holidays with start_date and duration                                                                                                                                                                                                     | holidays        |
| `request`            | All request types in one table (discriminated by request_type_id); fields: request_date, compensation_date, start_date_time, end_date_time, off_time_hour, reason_id, status, approve_by, confirm_by, error_count, manager_comment, admin_comment | requests        |
| `request_type`       | Reference: forget / leave / compensation (late-early)                                                                                                                                                                                             | requests        |
| `reason`             | Reference: leave reasons with max_request_day                                                                                                                                                                                                     | leave           |
| `user_leave`         | Annual leave balance per user per year: total_leave, total_remain, carry_over, carry_over_remain                                                                                                                                                  | leave           |
| `member_schedule`    | RRule-based WFH/office pattern per member (active flag, schedule string)                                                                                                                                                                          | member-schedule |
| `user`               | Employee profiles — **NOT in scope for time module** (belongs to people)                                                                                                                                                                          | people          |
| `role` / `user_role` | Role assignments — **NOT in scope** (belongs to kernel/identity)                                                                                                                                                                                  | kernel          |
