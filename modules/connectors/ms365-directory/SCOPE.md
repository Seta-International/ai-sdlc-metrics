# SCOPE — modules/connectors/ms365-directory  (@seta/connector-ms365-directory)

## Purpose

Microsoft 365 Directory (Entra ID) vendor adapter. Owns the `users` + `groups` + `group_members` mirror in schema `connector_ms365_directory`, the delta-token sync state, the `ConnectorDefinition` manifest, and the JIT mapper that turns inbound Entra ID-token claims into rows in `auth.users` + `directory.external_identities` (via `@seta/directory`). Read-only against Graph (`capabilities.writes: false`); writes happen only to local cache and the canonical directory tables. (setup.md §7, §11 directory section.)

## Responsibilities

- **Owns:**
  - The `directoryConnector: ConnectorDefinition` manifest — id `ms365-directory`, providerId `entra`, delegated `User.Read`, application `User.Read.All` + `Group.Read.All`, `capabilities: { syncable: true, writes: false }` (current `src/manifest.ts`).
  - Postgres schema `connector_ms365_directory` and its migrations — tables `directory_users`, `directory_groups`, `directory_group_members`, `sync_state` (current `src/schema.ts`, `migrations/0000_tan_kingpin.sql`, `drizzle.config.ts` with `schemaFilter: ['connector_ms365_directory']`).
  - The JIT mapper: ID-token claims (`oid`, `tid`, `upn`, `email`, `name`) → upsert into `auth.users` + `directory.external_identities` via `@seta/directory` (setup.md §11 `jit-mapper.ts`).
  - Delta-sync mechanics — `sync_state.delta_token`, `last_full_sync_at`, `last_delta_sync_at`, `status` per resource kind (current `src/schema.ts`).
  - Drizzle row types: `DirectoryUser`, `DirectoryGroup`, `DirectoryGroupMember` (`$inferSelect`, current `src/schema.ts`).

- **Does NOT own:**
  - Session creation, cookie issuance, inbound SSO web UI — that is P2 `@seta/identity` (setup.md §11 P2 list).
  - The canonical `auth.users` / `directory.external_identities` tables themselves — those are owned by `@seta/auth` and `@seta/directory` respectively; this connector only writes through their interfaces (setup.md §11 platform/auth + platform/directory).
  - Writes back to Graph — manifest pins `writes: false`. If a future capability needs to write users/groups, it is a separate connector or a manifest-flag change with an ADR.
  - Tools, agent runtime, channel transport — boundary rules (CLAUDE.md; setup.md §11).
  - Cross-context FKs to `connector_ms365_planner.*` — references are by `entra_object_id` text only (CLAUDE.md "No cross-schema foreign keys").

## Public interface

All exports from `modules/connectors/ms365-directory/src/index.ts`.

- `directoryConnector: ConnectorDefinition` — manifest registered in `apps/api/src/main.ts` (setup.md §11 composition example; current `src/manifest.ts`).
- Drizzle schema re-exports: `connectorMs365Directory` (pgSchema), `directoryUsers`, `directoryGroups`, `directoryGroupMembers`, `syncState` (current `src/schema.ts`).
- Row types: `DirectoryUser`, `DirectoryGroup`, `DirectoryGroupMember` via `$inferSelect` (current `src/schema.ts`; CLAUDE.md "Drizzle table → row types").
- `mapIdTokenToUser(claims: EntraIdTokenClaims) => Promise<auth.users row>` — JIT mapper; calls `@seta/directory` to upsert `external_identities` and `auth.users` (setup.md §11 `jit-mapper.ts`).
- `syncDirectoryDelta(opts: { tenantId: string, resource: 'users' | 'groups' | 'group_members' }) => Promise<SyncResult>` — pulls `@odata.deltaLink` from `sync_state`, drains via `@seta/ms-graph` pagination, upserts the cache, advances `delta_token`.
- `routes(): Hono` — **mandatory `routes(handler?: Handler) => Hono` export** per CLAUDE.md. May expose admin-only `POST /sync/users`, `POST /sync/groups` triggers, or return an empty router if sync is purely scheduled.

## Patterns to follow

- **Schema-per-module Drizzle layout** — `src/schema.ts` + `drizzle.config.ts` with `schemaFilter: ['connector_ms365_directory']` + `migrations/` directory; `@seta/db` runs migrations in dependency order (CLAUDE.md "Schema-driven"). Already in place; preserve it.
- **Generated migrations only** — never hand-edit `migrations/*.sql`; fix `schema.ts` and run `drizzle-kit generate` (CLAUDE.md).
- **`tenant_id uuid not null` on every persisted row** — already correct in `schema.ts`; PKs are tenant-scoped composites (CLAUDE.md "Every persisted row has `tenant_id`").
- **RLS policies on every tenant-data table** — `@seta/db` enforces; the app role `tenant_user` is RLS-gated and `platform_admin` (`bypassRls: true`) is ops-only (CLAUDE.md footguns).
- **Natural keys, not auto-increment** — `entra_object_id`, `entra_group_id` are the natural keys; `tenant_id` is the universal correlation key (CLAUDE.md "Idempotent external boundaries").
- **JIT mapping through `@seta/directory`** — never write `auth.users` directly; go through the directory interface so all IdPs converge on one mapper contract (setup.md §11 platform/directory description).
- **Delta sync via `sync_state.delta_token`** — full sync once, deltas thereafter; track `status` to surface stuck syncs (current `schema.ts`).
- **Connector consent gate** — call sites verify `connectorRegistry.requireConsent(tenantId, 'ms365-directory')`; the connector treats Graph 403 as `Forbidden` and surfaces a typed error (CLAUDE.md "Connector consent").
- **Tenant id from context, not parameter** — read via `tenantContext.getTenantId()` inside the connector; explicit `tenantId` params allowed only at the seam where a background worker constructs the call (CLAUDE.md "Tenant id is never a function parameter"; setup.md §11 dep direction).
- **OTel spans on every Graph call** via `@seta/ms-graph` middleware (setup.md §11).
- **`routes()` export** for uniform mounting even if it returns an empty Hono router (CLAUDE.md "Every `modules/*` package exports `routes`").
- **Errors throw `DomainError` subclasses from `@seta/middleware/errors`** (CLAUDE.md; setup.md §15).

## Patterns to avoid

- **Importing `@seta/agent`, `@seta/teams`, or any other `modules/products/*`/`modules/channels/*` package** — boundary violation (CLAUDE.md; setup.md §11).
- **Hand-editing `migrations/*.sql`** — regenerate via `drizzle-kit generate` (CLAUDE.md).
- **`drizzle-kit push` against shared DBs** — local-dev only (CLAUDE.md footguns).
- **Cross-schema FKs** — reference `connector_ms365_planner.*` rows or `auth.users` only by id, never via FK (CLAUDE.md).
- **Writing `auth.users` rows directly** — go through `@seta/directory`'s mapper interface (setup.md §11).
- **Using the official `@microsoft/microsoft-graph-client` SDK** — explicit non-pick; use `@seta/ms-graph` (setup.md §11 §13).
- **`process.env.X` outside `drizzle.config.ts`** — read typed env via `@seta/api`'s boot (CLAUDE.md "`process.env` → typed `env`"). The `dotenv` import here is scoped to the kit config only.
- **Maintaining a parallel `interface` for Drizzle rows** — use `$inferSelect`/`$inferInsert` (CLAUDE.md).
- **Mocking `@seta/ms-graph`, `@seta/oauth`, or `@seta/directory`** in tests — never mock internal `@seta/*` modules; use msw recordings (CLAUDE.md).
- **Calling `openai`/`@anthropic-ai/sdk`** — no LLM in connectors.

- **Unit (`src/**/*.test.ts`):**
  - Manifest contract (already present — `manifest.test.ts`).
  - JIT mapper claim → row shape (table-driven against canonical Entra ID-token payloads).
  - Delta-token state machine — happy path, recovery from `status='error'`.
- **Integration (`tests/integration/**`, requires `DATABASE_URL`):**
  - Migrate `connector_ms365_directory`; run a full + delta sync against msw-recorded Graph fixtures; assert RLS isolation between tenants.
  - JIT mapper writes through `@seta/directory` against a real `auth.users` table.
- **External HTTP via msw recordings only** — `graph.microsoft.com/v1.0/users`, `groups`, delta endpoints (CLAUDE.md; setup.md §17). Recordings checked into `__recordings__/`.
- **No live Graph; no `vi.mock` of internal `@seta/*`** (CLAUDE.md).
